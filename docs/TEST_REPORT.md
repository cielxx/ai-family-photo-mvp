# AI 家庭纪念影像助手 MVP 测试记录

## 测试目标

验证 v0.1 Streamlit Demo 的核心主流程是否可用于作品集演示，包括初次处理 Workflow、半自动上传结果图、结果展示、二次调整 Workflow、二次调整结果上传和重新开始等关键路径。

## 测试环境

- 测试时间：YYYY-MM-DD
- 测试版本：v0.1 开发阶段收尾版
- 测试方式：人工手动测试
- 操作系统：macOS
- 运行方式：本地 Streamlit
- Python 环境：项目 `.venv` 虚拟环境
- 前端框架：Streamlit
- AI 编排：Dify Workflow
- Dify 状态：已配置真实初次处理 Workflow 与二次调整 Workflow
- 图像处理：v0.1 采用人工 / 半自动处理后上传结果图

## 测试范围

本轮测试覆盖：

- 三类服务的初次处理入口
- 上传照片与授权校验
- 初次 Dify Workflow 调用后的页面流转
- 初次效果图上传与结果页展示
- 二次调整需求校验
- 二次调整 Dify Workflow 调用后的页面流转
- 二次调整效果图上传与调整完成页展示
- 重新开始状态清理

不覆盖：

- 真实图像处理 API
- 登录、支付、订单、历史任务
- 多轮无限调整
- 移动端原生交互
- 生产环境性能和安全渗透测试

## 已知限制与后续验证项

- v0.1 暂未接入真实图像处理 API，图像生成质量不在本轮自动化验证范围内。
- 半自动处理依赖操作员根据 Prompt 使用外部工具完成图片处理，后续需验证真实图像 API 接入后的端到端稳定性。
- 当前不支持多轮无限调整，后续如扩展多轮调整，需要补充版本管理、任务链路和回退测试。
- 当前不支持登录、订单、支付、历史任务，后续如进入产品化阶段需要补充账号、数据存储和权限测试。
- Streamlit Demo 主要用于作品集演示，后续如面向移动端用户，需要补充移动端布局和交互测试。
- Dify Workflow 详情仅用于内部演示、调试和半自动处理，截图或公开展示时需避免暴露敏感配置和内部信息。
